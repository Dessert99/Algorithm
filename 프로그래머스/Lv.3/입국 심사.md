```javascript
function solution(n, times) {
    let answer = 0;

    // 입국 심사관 수
    const immigrationOfficer = times.length;

    // 심사 받는데 걸리는 시간 양 끝 기준점
    let left = 1;
    // 심사관 한 명이 모든 사람을 심사할 때 걸리는 시간
    let right = Math.max(...times) * n; // 심사 시간이 가장 긴 사람이 n명의 사람을 모두 심사할 때

    // 이분 탐색 시작
    while (left <= right) {
        // 중간값: 현재 심사를 시도해보는 시간
        const mid = Math.floor((left + right) / 2);

        // mid 시간 동안 몇 명이나 심사할 수 있는지 계산한다.
        let totalImmigratedPeople = 0; // 최종적으로 심사받은 사람 수

        for (let time of times) {
            // 각 심사관은 동시에 심사할 수 있다.
            totalImmigratedPeople += Math.floor(mid / time);
        }

        // 만약 mid 시간 안에 n명이 모두 심사 받았다면
        if (totalImmigratedPeople >= n) {
            // 일단 정답으로 가정
            answer = mid;

            // 최솟값을 구해보자
            right = mid - 1;
        } else {
            // 아직 n 명을 모두 심사하지 못하는 경우
            left = mid + 1;
        }
    }

    return answer;
}

console.log(solution(6, [7, 10]));
```
### 🔗 풀이
1. `left`, `right` 기준점을 설정한다.
    - 이때 `right`는 심사관 한 명이 모든 사람을 심사할 때 걸리는 시간이다.
2. 이분 탐색을 시작한다. `while(left<=right)`
3. 중간값을 구한다. 이때 중간값은 “현재 심사를 시도해보는 시간”을 의미한다.
4. `mid` 시간 동안 몇 명이나 심사할 수 있는지 계산한다. (`totalImmigratedPeople`)
5. 심사관은 동시에 심사할 수 있으니, `times`배열을 순회하며 누적합을 구한다.
6. 만약 `totalImmigratedPeople` 이 `n`  이상이라면 `mid` 는 조건을 만족한 시간이다. 일단 저장해놓고, 범위를 좁혀가며 최솟값을 찾아본다.

### 🔗 배운점
1. 스프레드 연산자 내부 동작
    
    > 스프레드 연산자가 펼친 10만 개의 요소는 호출 스택에 들어가게 되는데, 너무 많은 요소가 한 번에 몰리면 오류가 날 수도 있다. `RangeError: Maximum call stack size exceeded`
    > 
    
    ```jsx
    ❌ Math.max(...times)
    ```
    
    1. 배열 `times`를 메모리에서 꺼낸다.
    2. `...` 연산자가 배열의 요소를 하나하나 쪼개서 스택(Stack)에 쌓는다.
    3. 마치 우리가 코드로 `Math.max(t[0], t[1], t[2], ... t[99999])`라고 10만 개를 타이핑한 것과 똑같이 변환된다.
    4. 브라우저마다 다르지만, 보통 인자 개수가 65,536개($2^{16}$) 를 넘어가면 `RangeError: Maximum call stack size exceeded` 가 발생할 위험이 있다.
    - 해결 방안
        
        ```jsx
        // 1. 정렬하고
        times.sort((a,b) => a-b)
        
        // 2. 가장 큰 값을 사용한다.
        times[times.length-1]
        ```
        
2. 자바스크립트 숫자 자료형 정리
    
    > 숫자가 크다면 `Number`가 아닌 `Bigint`를 써야 한다.
    > 
    
    | **특징** | **Number (기본 숫자)** | **BigInt (큰 정수)** |
    | --- | --- | --- |
    | **정의** | 64비트 부동소수점 (IEEE 754) | 임의 정밀도 정수 (Arbitrary-precision) |
    | **안전한 최댓값** | **약 $9,000$조** ($2^{53} - 1$)
    `9,007,199,254,740,991` | **제한 없음**
    (메모리가 허용하는 한 무한대) |
    | **소수점 표현** | 가능 (`3.14`) | **불가능** (정수만 표현, 소수점 버림) |
    | **사용법** | `10`, `3.5` | `10n`, `BigInt(10)` |
    | **이 문제에서의 한계** | 최악의 경우($10^{18}$, 100경)를 담을 수 없음.
    오차가 발생하여 계산이 틀림. | $10^{18}$은 가볍게 처리 가능.
    **오차 없이 정확함.** |
    | **주의사항** | 정밀도가 떨어지는 구간이 존재함 | `Number` 타입과 섞어서 연산 불가 (`10n + 5` ❌ → `10n + 5n` ⭕) |
