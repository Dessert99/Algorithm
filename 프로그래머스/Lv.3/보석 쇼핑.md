```javascript
function solution(gems) {
    // 전체 종류 파악
    const totalTypes = new Set(gems).size
    const map = new Map()
    
    let left = 0
    
    // 초기값으로 최대 길이
    const answer = [0, gems.length-1]
    
    // right무조건 이동
    for (let right=0;right<gems.length;right++){
        // right 요소
        const curGem = gems[right]
        
        // 보석 추가
        map.set(curGem, map.has(curGem) ? map.get(curGem)+1: 1)
        
        // 왼쪽 보석이 윈도우 내 중복되었다면 이동하면서 제거하기
        while(map.get(gems[left]) >=2){
            map.set(gems[left], map.get(gems[left])-1)
            left++
        }
        
        // map에 전부 모였다면 윈도우 크기 갱신
        if (map.size === totalTypes){
            // 직전 윈도우 크기
            const lastLength = answer[1] - answer[0]
            // 지금 윈도우 크기
            const curLength = right - left
            
            if (curLength<lastLength){
                answer[0] = left
                answer[1] = right
            }
        }
    }
    
    
    return [answer[0]+1, answer[1]+1] ;
}
```
### 🔗 사고 과정
1. 🤔 구현 방법 예상
    1. 투 포인터
    2. 해시맵

2. 🤔 주의사항
    - 인덱스와 진열대 번호가 다르다.
    - gems의 길이는 최대 10만이라서 이중 순회하면 초과난다.

3. 🤔 ❌ gems의 어떤 범위 내에 모든 종류의 보석이 있다는 것을 어떻게 판단할까?
    1. 일단 최소 윈도우 크기를 정한다. -> map크기만큼.
    2. gems처음부터 시작한다.
    3. 윈도우 내 요소들을 순회하며 map에 전부 있는지 확인한다.
    4. 없다면 right을 1 증가시킨다.
    5. 만약 gems[right]가 gems[left]와 같은 요소라면, left도 증가시킨다.
        - 근데 윈도우 안에 gems[left]가 이미 있었다면, gems[right]에 나오지 않아도 옮겨도 된다.
    - 이 방법은 틀렸다. 매번 Set을 돌리는 작업은 O(N)을 유발한다.

4. ⭐️ 새로운 발상
    1. 공용으로 관리하는 Map을 만들고
    2. 모든 종류가 Map에 들어올 떄까지 right를 옮기고
    3. left가 가리키는 보석이 현재 윈도우에서 2개 이상일 때만 우측으로 옮긴다.
    - 슬라이딩 윈도우는 "right는 무조건 이동", "left는 조건에 맞을 때 이동"이다. 이전 방법은 right도 조건에 따라 움직였다.

### 🔗 배운점
1. 슬라이딩 윈도우에서 `right`는 무조건 움직여야 한다.
2. 매 반복문에서 `Set()`연산은 비용이 크다.
