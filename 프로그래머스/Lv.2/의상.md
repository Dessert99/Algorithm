```javascript
function solution(clothes) {
    var answer = 1;

    const map = new Map();

    // 옷을 해시맵에 담기
    clothes.forEach((item) => {
        const type = item[1]; //타입만 사용

        // 만약 기존에 있는 타입이라면, + 1 해준다.
        map.set(type, map.get(type) + 1 || 1);
    });

    // 맵을 순회하며 각 옷의 개수에 + 1을 한 뒤 마지막에 1을 뺀다.
    for (const [key, value] of map) {
        answer *= value + 1;
    }

    return answer - 1;
}
```
### 🔗 배운점
1. 일일이 옷을 입혀보는 단순한 방식보다, 수학적으로 접근하는 습관을 들이자.
2. 이 문제에서 규칙은 어느 부분에서 근거를 찾을 수 있을까?
  - ⭐️ 모자를 고르는 행위가 안경을 고르는 행위에 영향을 주지 않는다. 이런식으로 선택지가 독립적일 경우, 곱의 법칙을 사용할 수 있다.
3. 해시맵 순회하기
    ```javascript
      for (const [key, value] of map) {
      console.log(key, value)
        }
    ```
4. 직관적인 코드로 개선
   ```javascript
   map.set(type, map.get(type) + 1 || 1);
   ```
   1. 만약 맵에 없는 타입이라면 map.get(type) 값이 undefined 가 된다.
   2. undefined에 숫자 연산을 하게 되면 NaN이 출력된다.
   3. NaN은 falsy한 값이기 때문에 논리 연산자 || 동작에 의해 오른쪽 값이 채택된다.
   - 좀 더 직관적인 코드는 다음과 같다.
     ```javascript
      map.set(type, (map.get(type) || 0) + 1);
     ```
