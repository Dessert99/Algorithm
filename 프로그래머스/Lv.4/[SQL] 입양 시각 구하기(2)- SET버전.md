```sql
-- 1. 변수 초기화 (-1부터 시작, 첫 행에서 0이 되도록 설정)
SET
    @hour := -1;

SELECT 
    (@hour := @hour + 1) AS HOUR,
    (
        SELECT 
            COUNT(ANIMAL_ID)
        FROM
            ANIMAL_OUTS
        WHERE
            HOUR(DATETIME) = @hour
    ) AS COUNT
FROM
    ANIMAL_OUTS
WHERE
    @hour < 23 -- 23이 될 때까지
```
### 🔗 풀이

### 🔗 배운점
1. `SET`이란?
   - "내 마음대로 쓸 수 있는 변수를 만들고, 거기에 값을 저장(초기화)하겠다"고 선언하는 명령어
2. 사용자 정의 변수: `@`
   - MySQL에서는 변수 이름 앞에 반드시 `@`를 붙여서 "이것은 테이블의 컬럼이 아니라, 내가 만든 변수다"라는 것을 표시한다.
   - 이렇게 만든 변수는 데이터베이스에 접속해 있는 동안(현재 세션) 메모리에 임시로 저장되어 계속 유지된다.
3. `:=` 와 `=`의 차이점
   - `SET` 명령어 뒤에서는: `SET @hour = -1;` 처럼 `=`을 써도 값을 넣는 역할(대입)을 합니다. `:=`를 써도 똑같다.
   - `SELECT` 명령어 안에서는: `=`기호를 쓰면 MySQL은 이를 값을 넣으라는 뜻이 아니라 "왼쪽과 오른쪽이 같은가?"를 묻는 비교 연산자로 해석해 버린다.
   - 결론: 그래서 `SELECT` 문 안에서 변수에 값을 더하고 덮어씌우려면, 비교 연산자와 헷갈리지 않도록 반드시 `:=` (대입 연산자)를 사용해야 한다. 실수를 방지하기 위해 `SET`을 할 때부터 아예 `:=`를 쓰는 습관을 들이는 경우가 많다.
4. `SET`동작 과정
    1. 준비 단계 (`SET @hour := -1;`)
        - 쿼리가 본격적으로 실행되기 전, MySQL 메모리 한 켠에 `@hour`라는 이름의 포스트잇을 붙이고 -1이라고 적어둡니다. (출발선 설정)
    2. 첫 번째 행 스캔 (`SELECT (@hour := @hour + 1) ...`)
        - `ANIMAL_OUTS` 테이블의 첫 번째 데이터를 읽습니다.
    3. `SELECT` 절을 만나면 포스트잇(`@hour`)에 적힌 값(-1)을 꺼내 1을 더합니다.
        - 결과는 0이 되고, 포스트잇의 숫자를 지운 뒤 0으로 고쳐 씁니다. (이때 0시의 입양 건수를 센 서브쿼리가 같이 실행됩니다.)
    4. 두 번째 행 스캔
        - 테이블의 두 번째 데이터를 읽습니다.
        - 다시 포스트잇 값을 꺼냅니다. 지금 적힌 값은 0입니다. 여기에 1을 더해 1로 고쳐 씁니다. (1시의 입양 건수 계산)
    5. 반복 및 종료 (`WHERE @hour < 23;`)
      - 이 과정을 테이블의 데이터 개수만큼 계속 반복하다가, 포스트잇(`@hour`)에 적힌 숫자가 23이 되는 순간 `WHERE` 조건에 걸려 반복을 멈추고 결과를 출력합니다.
5. `SET`으로 새로운 테이블 만들기
   ```sql
      -- 1. 변수 초기화 (-1부터 시작, 첫 행에서 0이 되도록 설정)
      SET
          @hour := -1;
      
      SELECT 
          (@hour := @hour + 1) AS HOUR
      FROM
          ANIMAL_OUTS
      WHERE
          @hour < 23 -- 23이 될 때까지
   ```
      | HOUR |
      |-----:|
      | 0 |
      | 1 |
      | 2 |
      | ... |
      | 23 |
     - 단, `ANIMAL_OUTS`에 행이 24개 이상이어야 한다. 만약 해당 테이블에 행이 5개라면, HOUR는 0~4까지만 나온다.
