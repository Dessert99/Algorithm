[https://leetcode.com/problems/consecutive-numbers/description/](https://leetcode.com/problems/consecutive-numbers/description/)
```sql
WITH
    SUB AS (
        SELECT
            num,
            -- 직전 행
            LAG(num, 1) OVER (ORDER BY id) AS prev,
            -- 2행 전
            LAG(num, 2) OVER (ORDER BY id) AS prev2
        FROM
            Logs
    )

SELECT
    -- 이 문제 한정으로 DISTINCT가 필요하다. 4연속 행이 나올 경우, 1이 두 번 집계되기 떄문이다.
    DISTINCT num AS ConsecutiveNums
FROM
    SUB
WHERE
    -- 3연속 나타나는 숫자
    num = prev AND
    prev = prev2
```
### 🔗 풀이

### 🔗 배운점
1. `LAG` 기본 문법
   > 현재 행을 기준으로 이전 행의 데이터를 가져올 때 사용한다.
   ```sql
     LAG(컬럼명, 오프셋, 기본값) OVER (ORDER BY 정렬기준)
   
      -- 직전 행
     LAG(num, 1) OVER (ORDER BY id) AS prev,
      -- 2행 전
     LAG(num, 2) OVER (ORDER BY id) AS prev2
   ```
   - 컬럼명: 가져오고 싶은 데이터가 있는 컬럼입니다.
   - 오프셋 (선택): 몇 칸 이전의 데이터를 가져올지 정합니다. (기본값은 1)
   - 기본값 (선택): 이전 행이 없을 때(예: 첫 번째 행) 대신 보여줄 값입니다.
   - `OVER (ORDER BY ...)`: 어떤 순서로 줄을 세워 "이전"을 정의할지 정합니다. 매우 중요합니다.
2. 그림으로 확인하기
   - 기본 데이터
      | id | num |
      | --- | --- |
      | 1 | 1 |
      | 2 | 1 |
      | 3 | 1 |
      | 4 | 2 |
      | 5 | 1 |
      | 6 | 2 |
      | 7 | 2 |
    - `LAG`로 만든 테이블 상태  
      | **(id)** | **num (현재)** | **prev (1행 전)** | **prev2 (2행 전)** | **비고** |
      | --- | --- | --- | --- | --- |
      | 1 | **1** | NULL | NULL | 이전 데이터 없음 |
      | 2 | **1** | 1 | NULL | 2행 전 데이터 없음 |
      | 3 | **1** | **1** | **1** | ✨ **조건 만족 (num=prev=prev2)** |
      | 4 | **2** | 1 | 1 | num이 달라짐 |
      | 5 | **1** | 2 | 1 |  |
      | 6 | **2** | 1 | 2 |  |
      | 7 | **2** | 2 | 1 |  |
    - `id=3`행을 보면 `num`, 직전값, 두 칸 전 값이 전부 1로 동일하다.
3. 이 문제에서 `DISTINCT`가 필요한 이유
   - 같은 숫자가 3번을 넘어 4번, 5번 연속으로 나올 경우 중복된 결과가 출력되기 때문이다.
   - 기본 테이블
     | **id** | **num** |
      | --- | --- |
      | 1 | 1 |
      | 2 | 1 |
      | 3 | 1 |
      | 4 | 1 |
      | 5 | 2 |
   - `LAG`로 만든 테이블
     | **id** | **num** | **prev** | **prev2** | **조건 만족? (num=prev=prev2)** |
      | --- | --- | --- | --- | --- |
      | 1 | 1 | NULL | NULL | X |
      | 2 | 1 | 1 | NULL | X |
      | **3** | **1** | **1** | **1** | **O (첫 번째 매칭)** |
      | **4** | **1** | **1** | **1** | **O (두 번째 매칭)** |
      | 5 | 2 | 1 | 1 | X |
       - id가 3일 때도 조건을 만족하고, 4일 때도 조건을 만족한다.
   - `DISTINCT`를 사용하지 않는 경우
     | ConsecutiveNums |
     | --- |
     | 1 |
     | 1 |
       - `id=3`에서 한 번, `id=4`에서 한 번. 총 두 번이 나온다.
