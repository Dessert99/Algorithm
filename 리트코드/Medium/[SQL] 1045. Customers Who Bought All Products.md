[https://leetcode.com/problems/customers-who-bought-all-products/description/](https://leetcode.com/problems/customers-who-bought-all-products/description/)
```sql
WITH
    -- 중복을 제거한 customer_id, product_key를 반환
    RESULT AS (
        SELECT DISTINCT
            customer_id,
            product_key
        FROM
            Customer 
    )

SELECT
    customer_id
FROM
    RESULT
GROUP BY
    customer_id -- 고객 ID기준으로 그룹화하면 제품 키 정보가 뭉쳐진다.
HAVING
    COUNT(product_key) = (
        -- Product 키 갯수
        SELECT
            COUNT(*)
        FROM
            Product
    )
```
### 🔗 풀이
1. 💡 문제 분석
    1. Customer테이블에 있는 모든 제품을 구매한 고객의 ID

2. 🤔 만약 JOIN한다면
    - 이 테이블에는 중복 행이 포함될 수 있습니다 -> 중복된 행이 있을 수 있어서 제거해야 한다.
    - distinct

### 🔗 배운점
1. DISTINCT는 윈도우 함수 이후에 실행한다.
    - 그래서 원하는 대로 실행이 안 될 수도 있음.
2. 무리하게 윈도우 함수를 쓰려고 하지 말자. 윈도우 함수마다 패턴이 있는데, 그 키워드가 보일 때 사용을 고려하자.
   | **카테고리** | **핵심 키워드 (이 말이 들리면 소환!)** | **대표 함수** |
    | --- | --- | --- |
    | **순위 매기기** | "가장 ~한 상위 N개", "등수", "순번", "공동 순위 처리" | `ROW_NUMBER`, `RANK`, `DENSE_RANK` |
    | **이전/다음 비교** | "전날 대비", "지난달과 비교", "증감률", "다음 로그와의 간격" | `LAG`, `LEAD` |
    | **누적/흐름** | "누적 합계", "현재까지의 총합", "7일 이동 평균" | `SUM() OVER`, `AVG() OVER` |
    | **처음과 끝** | "최초 구매일", "마지막 접속 기록", "범위 내 첫 값" | `FIRST_VALUE`, `LAST_VALUE` |
    | **비율/분포** | "상위 10%", "4등분(쿼타일)", "백분위" | `PERCENT_RANK`, `NTILE` |
