[https://leetcode.com/problems/exchange-seats/description/](https://leetcode.com/problems/exchange-seats/description/)
```sql
WITH
    result as (
        SELECT
            id-1 AS id_,
            student
        FROM
            Seat
        WHERE
            id % 2 = 0 -- 짝수 행만 뽑기

        UNION ALL

        SELECT
            id+1 AS id_,
            student
        FROM
            Seat
        WHERE
            id % 2 =1 -- 홀수 행만 뽑기
    )

SELECT
    ROW_NUMBER() OVER(ORDER BY id_) AS id, -- 마지막 홀수 번호를 위해서 새로운 번호 매기기
    student
FROM
    result
```
### 🔗 풀이

### 🔗 배운점
1. 행을 홀/짝으로 구분하여 두 개의 테이블을 만들고 union으로 합치는 방식
    1. CTE생성
    2. 홀수 id만 뽑아서 테이블 생성
    3. 짝수 id만 뽑아서 테이블 생성
    4. union으로 합치기 (union all 가능)
     - BEFORE
       | id | student |
        | --- | --- |
        | 1 | Abbot |
        | 2 | Doris |
        | 3 | Emerson |
        | 4 | Green |
        | 5 | Jeames |
     - AFTER
        | id | student |
        | --- | --- |
        | 2 | Doris |
        | 4 | Green |
        | 1 | Abbot |
        | 3 | Emerson |
        | 5 | Jeames |
2. 가상 테이블 만들 때 `id-1`, `id+1` 하는 이유?
     | id-1 | student |
      | --- | --- |
      | 1 | Doris |
      | 3 | Green |
      | 2 | Abbot |
      | 4 | Emerson |
      | 6 | Jeames |
     - 마지막에 `id-`기준으로 정렬하면 홀/짝 위치가 바뀌게 된다.
3. `id_`컬럼이 있지만 새로운 `id`를 만드는 이유?
   - 데이터가 홀수일 경우 마지막 행이 5가 아니라 6이 된다. 학생 수가 홀수일 때, 마지막 학생은 자리를 바꾸지 않는다는 조건이 있다.
     | id_ | student |
      | --- | --- |
      | 1 | Doris |
      | 2 | Abbot |
      | 3 | Green |
      | 4 | Emerson |
      | 6 | Jeames |
     - 그래서 `ROW_NUMBER()` 로 새로운 번호를 생성하고 그걸 `id`로 정한다.
