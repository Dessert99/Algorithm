[https://leetcode.com/problems/product-price-at-a-given-date/description/](https://leetcode.com/problems/product-price-at-a-given-date/description/)
```sql
WITH
    -- 일단, 초기 가격 10을 붙여줘야 한다.
    new_table AS (
        SELECT
            *
        FROM
            -- 원래 테이블에
            Products
        UNION ALL
        -- 중복을 제거한 모든 id에 기본값 10을 붙여서 UNION한다.
        SELECT DISTINCT
            product_id,
            10 AS new_price,
            '1001-01-01' AS change_date
        FROM
            Products
    ),
    result AS (
        SELECT
            product_id,
            new_price AS price,
            -- id기준으로 파티션을 나눈 뒤, 날짜 내림차순으로 정렬한다.
            RANK() OVER (PARTITION BY product_id ORDER BY change_date DESC ) AS rnk
        FROM    
            new_table
        WHERE
            -- 2019-08-16 이전 상품만 필터링
            change_date <= '2019-08-16'
    )
SELECT
    product_id,
    price
FROM
    result
WHERE
    rnk = 1
```
### 🔗 풀이
1. 💡 문제 분석
    1. `change_date가 2019-08-16` 이전 날짜 기준으로
    2. 가장 나중에 바뀐 값이 최종 상품의 가격이다.

2. 👀 방법1)
    1. `change_date <= 2019-08-16` 이하인 행만 필터링해서
    2. `product_id`기준으로 `PARTITION BY`한 다음
    3. `change_date` 기준으로 `ORDER BY DESC`해서
    4. `RANK`를 매긴 다음
    5. 본 쿼리에서 `RANK = 1`인 행만 골라서 조회한다.
    6. 근데, 랭킹 테이블에 `product_id`가 없을 수도 있다. 이러면 기본값인 10을 넣어야 한다. -> `UNION`으로 초기값을 넣은 새로운 테이블 만들기
