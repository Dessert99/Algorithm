[https://leetcode.com/problems/user-activity-for-the-past-30-days-i/description/](https://leetcode.com/problems/user-activity-for-the-past-30-days-i/description/)
```sql
WITH
    distinct_table AS (
    SELECT DISTINCT
    user_id,
    DATE_FORMAT(activity_date, '%Y-%m-%d') AS activity_date

FROM
    Activity

WHERE
        -- 27일 포함 30일 전이어야 한다.
        DATEDIFF('2019-07-27', activity_date) < 30 AND
        -- activity_date가 미래라면, 음수가 나온다. 0이상 조건을 꼭 넣어줘야 한다.
        DATEDIFF('2019-07-27', activity_date) >= 0
    )   

SELECT
    activity_date AS day,
    COUNT(activity_date) AS active_users

FROM
    distinct_table
GROUP BY
    activity_date

```
### 🔗 사고 과정
1. 💡 문제 분석
    1. 2019-07-27포함 30일 이내.
    2. 1세션 -> 1유저
    3. 중복 행 존재

2. 👀 방법
    1. activity_date 기준 그룹화 -> DATE_FORMAT('%Y-%m-%d') 필수
    2. user_id 중복 제거 필수

### 🔗 배운점
1. `DATEDIFF(end, start)`
   - 주의사항
     - `start`가 `end`보다 클 경우, 결과는 음수가 나온다. 이 문제 같은 경우에 2019-07-27 포함 30일 전이어야 한다. 그래서 0이상 조건을 꼭 붙여줘야 한다.
