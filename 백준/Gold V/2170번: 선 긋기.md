[https://www.acmicpc.net/problem/2170](https://www.acmicpc.net/problem/2170)
```javascript
const input = require('fs')
    .readFileSync(process.platform === 'linux' ? '/dev/stdin' : './input.txt')
    .toString()
    .trim()
    .split('\n');

let inputIndex = 0;

// 선을 그은 횟수
const N = Number(input[inputIndex++]);

const arr = [];
for (let i = 0; i < N; i++) {
    arr.push(input[inputIndex++].split(' ').map(Number));
}

// 1. 정렬
arr.sort((a, b) => a[0] - b[0]);
const result = [arr[0]]; // 초기 세팅

// 2. 스위핑
for (let i = 1; i < N; i++) {
    // 현재 선분
    const current = arr[i];

    // 가장 마지막 선분
    const last = result[result.length - 1];

    // case1. 두 선분이 겹칠 때 -> 연장
    if (current[0] <= last[1]) {
        // 이전 선분의 끝을 두 선분의 끝 중 가장 큰 값으로 변경
        last[1] = Math.max(last[1], current[1]);
    } else {
        // case2. 두 선분이 떨어져 있을 때
        result.push(current); // 그냥 추가
    }
}

let answer = 0;
for (const line of result) {
    const [start, end] = line;

    answer += end - start;
}

console.log(answer);
```
### 🔗 풀이

### 🔗 배운점
1. 스위핑은 무엇을 위한 알고리즘인가?
   > 복잡하게 얽히고 겹쳐 있는 여러 개의 구간을 중복 없이 깔끔한 최소한의 덩어리로 합치는 것
   > 여기서 중요한 것은, 기본 뼈대는 정렬과 스캔이고, 스캔 중 일어나는 로직은 문제마다 다르다.
   - 예시 1) 일반적인 겹침
     
          ```
          before [[1, 5], [4, 8], [10, 12]] 
          after [[1, 8], [10, 12]]
          // 어차피 1~5와 4~8은 이어지니까, 결국 1부터 8까지 쭉 칠해져 있고, 중간에 띄어서 10부터 12까지 칠해져 있다.
          ```
    
    - 예시 2) 포함 관계
      
          ```
          before [[1, 10], [2, 6], [3, 8]]
          after [[1, 10]]
          // 자잘한 것들은 이미 1~10이라는 거대한 구간 안에 다 포함되므로, 결국 남는 건 1~10 하나뿐이다
          ```
2. 핵심 원리
      1. 정렬
          - 데이터를 특정 기준(주로 시작점의 x좌표 또는 시간)으로 오름차순 정렬한다.
      2. 스캔 
          - 기준선을 이동시키며(배열을 순회하며) 이전 상태와 현재 상태를 비교해 값을 갱신하거나 정답을 도출한다.
3. 코드 스니펫
   ```javascript
      function mergeIntervals(intervals) {
        if (intervals.length === 0) {
      			 return [];
      			}
      
        // 1. 시작점(x좌표)을 기준으로 오름차순 정렬 (스위핑의 가장 중요한 사전 작업)
        intervals.sort((a, b) => a[0] - b[0]);
        // 초기 세팅. 시작 기준점
        const result = [intervals[0]];
      
        // 2. 스위핑 시작
        for (let i = 1; i < intervals.length; i++) {
          const current = intervals[i]; // 빗자루가 지나가고 있는 현재 선분
          // 지금까지 병합해 놓은 결과물 중 가장 마지막 선분. 비교 대상이다.
          const lastMerged = result[result.length - 1];
      
          // case1. 두 선분이 겹칠 때 -> 연장
          if (current[0] <= lastMerged[1]) {
            // 이전 선분의 끝점을 두 선분의 끝점 중 더 큰 값으로 연장(갱신)
            lastMerged[1] = Math.max(lastMerged[1], current[1]);
          } else {
          // case2. 두 선분이 떨어져 있을 때 -> 새로 추가
            result.push(current); // 겹치지 않고 떨어져 있다는 의미
          }
        }
      
        return result;
      }
      
      // 테스트 실행
      const input = [[1, 3], [2, 6], [8, 10], [15, 18]];
      console.log(mergeIntervals(input));
      // 출력: [ [ 1, 6 ], [ 8, 10 ], [ 15, 18 ] ]
    ```
     - case1. 두 선분이 겹칠 때 예시
        - `lastMerged`가 [1, 3]이고, `current`가 [2, 6]인 상황에서 2는 3보다 작으니 겹친다.
        - 겹친다면 끝점을 더 멀리 뻗어나간 쪽으로 업데이트해 줍니다. 3과 6 중 더 큰 값(`Math.max`)인 6으로 끝점을 늘려준다.
        - 이제 `lastMerged`는 [1, 6]으로 병합되었다.
4. 시간복잡도
   - 이 알고리즘의 시간 복잡도는 배열을 정렬하는 데 걸리는 시간인 O(N log N)이 지배적이다. 정렬 후에는 배열을 단 한 번만 순회하므로 O(N)이 추가되어, 최종적으로 O(N log N)의 빠른 속도로 문제를 해결할 수 있다. 이중 반복문 O(N^2)을 써야 할 것 같은 문제를 획기적으로 줄준다.
