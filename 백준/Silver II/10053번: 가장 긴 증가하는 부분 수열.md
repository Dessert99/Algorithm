[https://www.acmicpc.net/problem/11053](https://www.acmicpc.net/problem/11053)
```javascript
const input = require('fs')
    .readFileSync(process.platform === 'linux' ? '/dev/stdin' : './input.txt')
    .toString()
    .trim()
    .split('\n');

// 수열 크기 (1~1000)
const N = Number(input[0]);

// 수열
const arr = input[1].split(' ').map(Number);

// dp[i] 의미: arr[i]를 마지막으로 했을 때의 최대 길이
const dp = Array.from({ length: N }, () => 1);

// arr 순회. (첫 번째는 넘어간다.)
for (let i = 1; i < N; i++) {
    // 현재 숫자
    const curNum = arr[i];

    // 처음부터 현재 위치(i-1)까지 확인. 내(i) 앞쪽(j)을 다 훑어본다
    for (let j = 0; j < i; j++) {
        const frontNum = arr[j];

        // 만약 앞쪽에 더 작은 숫자가 있다면?
        if (frontNum < curNum) {
            // 그 숫자의 dp값을 확인한다. dp값은 "그 숫자가 연결되어 있는 줄의 길이"이다.
            const frontDp = dp[j];

            //  현재 숫자의 dp값.
            const curDp = dp[i];

            // 앞쪽 길이에 현재 숫자를 붙였을 때 길이와, 현재 내 길이 중 큰 쪽으로 붙는다. 이때, curDp는 계속 바뀐다.
            // 이 부분 덕분에 [10, 50, 20, 30] 케이스를 해결할 수 있다.
            if (frontDp + 1 > curDp) {
                // 현재값을 그 뒷쪽에 붙임
                dp[i] = frontDp + 1;
            }
        }
    }
}
let answer = Math.max(...dp);

console.log(answer);
```
### 🔗 풀이
1. ⭐️ 단순히 마지막 숫자만 기억하면 안된다.
    - [10, 50, 20, 30] 케이스는 최대 길이가 3이지만, 마지막 숫자만 기억할 경우, [10, 50]에서 멈춰서 2가 출력된다.

2. 🤔 dp 배열에 어떤 데이터를 저장해야 할까?
    - dp[i] 의미: arr[i]를 마지막으로 했을 때의 최대 길이

### 🔗 배운점
1. `dp`배열에 넣을 정보를 선택하는 기준
   - "다음 단계와 연결될 수 있는 정보인가?"
       - 실패하는 정의: dp[i] = "$i$번째 인덱스까지의 숫자들 중 가장 긴 수열의 길이:
          -  문제점: 이렇게 정의하면, 그 수열이 어떤 숫자로 끝났는지 알 수가 없습니다. 다음 숫자(미래)가 내 뒤에 붙을 수 있는지 없는지(크기 비교) 판단할 수가 없죠.
       - 성공한 정의: dp[i] = "$i$번째 인덱스의 숫자를 반드시 마지막으로 포함하는 가장 긴 수열의 길이"
          - 이점: "아, 이 수열은 무조건 arr[i]로 끝나는구나!"라는 정보가 확정됩니다. 덕분에 다음 숫자들은 arr[i]와 대소 비교만 하면 연결 여부를 결정할 수 있습니다.
   -  "과거의 최적해가 누적되어 있는가?"
   -  "최종 정답을 어떻게 도출할 것인가?"
2. `dp`문제를 만났을 때 고려할 부분들
   - WHAT: dp[i]에 무엇을 저장할 것인가? (길이? 합? 개수?)
   - INTERFACE: dp[i]만 보면 다음 단계(i+1)를 계산할 때 필요한 정보(마지막 숫자, 남은 무게 등)가 충분한가?
   - SOURCE: dp[i]는 어디서 값을 물려받는가? (i-1에서? 아니면 0부터 i-1 전체에서?)
